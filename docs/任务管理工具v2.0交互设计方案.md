# 任务管理工具v2.0交互设计方案

## 一、核心交互设计原则

### 1. 无感升级原则
- **完全兼容**：新增功能对仅使用简单Todo的用户完全无感，不改变原有核心操作路径
- **零冗余**：不增加任何冗余步骤，保持原有操作流程的简洁性
- **平滑过渡**：新功能无缝集成到现有交互体系中，用户无感知

### 2. 渐进式披露原则
- **按需展示**：复杂功能仅在用户有需求时展示，默认隐藏
- **层次分明**：通过折叠/展开机制，将复杂功能组织成可管理的层级
- **干扰最小化**：不干扰主流程，做到「想用就有，不用不见」

### 3. 直觉化交互原则
- **符合习惯**：所有复杂功能的交互逻辑完全符合用户直觉，兼容行业通用操作习惯
- **零学习成本**：用户无需教程即可上手，操作逻辑自然流畅
- **反馈及时**：所有操作都有明确的视觉反馈，用户始终知道自己的操作结果

### 4. 视觉层级原则
- **清晰区分**：通过缩进、视觉标记、色彩区分等方式，清晰呈现不同层级的任务归属
- **信息可视化**：通过进度条、日历标记等方式，直观呈现任务状态和重复规则
- **避免过载**：合理组织信息，避免视觉混乱和信息过载

## 二、模块一：子任务体系交互设计

### 1. 多层级嵌套树结构交互

#### 核心设计
- **创建入口**：任务卡片展开时，在底部显示「添加子任务」按钮，默认隐藏
- **层级呈现**：通过缩进和视觉标记，清晰区分不同层级的任务
- **展开/收起**：每个任务卡片左侧添加展开/收起按钮，支持一键折叠所有子任务
- **操作兼容**：父子任务的编辑、删除、归档交互与原有单任务保持一致

#### 交互流程图
```
任务卡片 → 点击展开按钮 → 显示子任务列表 + 添加子任务按钮 → 点击添加子任务 → 弹出子任务输入框 → 输入子任务内容 → 确认添加 → 子任务显示在父任务下方（缩进显示）
```

#### 视觉设计
- **缩进规则**：每级子任务缩进16px，最大支持5级嵌套
- **视觉标记**：父任务左侧显示展开/收起图标，子任务左侧显示层级连接线
- **色彩区分**：子任务文字颜色比父任务稍浅，形成视觉层级
- **进度显示**：父任务右侧显示子任务完成进度条

### 2. 子任务拖拽排序交互

#### 核心设计
- **全场景兼容**：支持同层级排序拖拽、跨层级归属拖拽
- **实时反馈**：拖拽过程中通过ghost element展示任务落点，通过缩进引导线明确层级变化
- **吸附效果**：拖拽到层级边界时自动吸附，给出明确的视觉反馈
- **异常处理**：拖拽到无效区域时，给出明确的视觉提示，松手后自动回到原位置

#### 交互流程图
```
长按子任务 → 进入拖拽状态 → 移动到目标位置 → 显示ghost element和引导线 → 松手 → 任务移动到新位置
```

#### 视觉设计
- **拖拽状态**：拖拽时任务卡片半透明，显示ghost element
- **引导线**：拖拽过程中显示虚线引导线，指示层级变化
- **吸附反馈**：拖拽到边界时，引导线加粗，给出吸附提示
- **无效提示**：拖拽到无效区域时，显示红色禁止图标

### 3. 子任务独立状态管理交互

#### 核心设计
- **独立状态**：每个子任务配备独立的状态勾选框，与主任务完全分离
- **进度联动**：主任务区域实时展示子任务完成进度，通过进度条可视化呈现
- **状态提示**：子任务全量完成时，主任务给出对应的状态变化提示
- **详情页兼容**：子任务可单独打开详情页编辑，与主任务编辑逻辑完全一致

#### 交互流程图
```
点击子任务勾选框 → 子任务状态切换 → 父任务进度条更新 → 所有子任务完成时，父任务状态提示
```

#### 视觉设计
- **进度条**：父任务右侧显示小型进度条，实时反映子任务完成情况
- **状态提示**：所有子任务完成时，父任务卡片边缘显示绿色边框提示
- **勾选动画**：子任务勾选时，勾选图标有平滑的动画效果
- **详情页**：子任务详情页与主任务详情页保持一致的设计

## 三、模块二：重复任务体系交互设计

### 1. RRULE基础规则设置交互

#### 核心设计
- **渐进式面板**：默认仅展示核心频率选项（日/周/月/年），高级设置默认隐藏
- **口语化配置**：把专业的RRULE参数转化为口语化的选项，如「每2周的周一、周三」
- **实时预览**：配置过程中实时可视化预览重复规则，通过日历标记的方式展示重复日期
- **路径兼容**：用户可从原有日期选择面板一键进入重复规则设置

#### 交互流程图
```
任务详情页 → 点击日期选择器 → 选择日期后点击「重复」按钮 → 打开重复规则设置面板 → 选择频率 → 配置高级选项 → 查看日历预览 → 确认设置
```

#### 视觉设计
- **面板布局**：频率选择区域在上，高级设置区域默认折叠，点击展开
- **日历预览**：面板底部显示小型日历，重复日期用特殊颜色标记
- **口语化显示**：配置结果以口语化文本显示，如「每2周的周一、周三」
- **操作按钮**：「取消」「确定」按钮清晰可见，操作流程明确

### 2. 结束条件设置交互

#### 核心设计
- **同面板展示**：结束条件与重复规则设置同面板展示，采用选项卡切换
- **可视化预览**：每个结束条件配套清晰的说明与可视化预览
- **无缝切换**：切换结束条件时，无输入内容丢失，支持无缝切换
- **逻辑兼容**：兼容原有单任务的截止日期设置逻辑

#### 交互流程图
```
重复规则设置面板 → 切换到「结束条件」选项卡 → 选择结束条件类型 → 配置具体参数 → 查看预览 → 确认设置
```

#### 视觉设计
- **选项卡**：「重复规则」「结束条件」选项卡清晰可辨
- **条件类型**：单选按钮形式展示「永不结束」「重复次数」「结束日期」三个选项
- **预览区域**：每个选项下方显示对应的预览信息，如「将重复10次，预计结束于2023-12-31」
- **参数输入**：根据选择的条件类型，动态显示对应的输入控件

### 3. 例外日期管理交互

#### 核心设计
- **直接操作**：支持在日历视图、任务列表中，直接对单次重复任务进行例外操作
- **极简选项**：仅保留「跳过本次」「修改本次」两个核心选项
- **视觉标记**：例外日期在日历视图、任务列表中做明确的轻量标记
- **批量管理**：支持在重复规则面板中，统一查看、管理、取消所有例外日期

#### 交互流程图
```
任务列表/日历视图 → 长按重复任务 → 弹出例外操作菜单 → 选择「跳过本次」或「修改本次」 → 执行相应操作
```

#### 视觉设计
- **操作菜单**：长按任务后弹出简洁的操作菜单，仅包含两个核心选项
- **例外标记**：例外任务在列表中显示特殊标记，如虚线边框或特殊图标
- **日历标记**：日历视图中，例外日期用不同颜色或图标标记
- **管理界面**：重复规则面板中添加「例外日期」选项卡，显示所有已设置的例外日期

## 四、核心页面高保真线框图

### 1. 任务列表页面

#### 主要元素
- **任务卡片**：包含标题、描述、截止日期、优先级、子任务进度条、展开/收起按钮
- **子任务**：缩进显示，包含标题、状态勾选框
- **拖拽区域**：任务卡片左侧显示拖拽手柄
- **操作按钮**：编辑、删除按钮

#### 交互热区
- **展开/收起按钮**：点击展开/收起子任务列表
- **添加子任务按钮**：展开后显示，点击弹出子任务输入框
- **子任务勾选框**：点击切换子任务状态
- **拖拽手柄**：长按进入拖拽状态
- **任务卡片**：点击进入任务详情页

### 2. 任务详情页面

#### 主要元素
- **基本信息**：标题、描述、截止日期、优先级
- **重复设置**：重复开关、频率选择、结束条件设置
- **子任务管理**：子任务列表、添加子任务按钮
- **操作按钮**：保存、删除、取消

#### 交互热区
- **重复开关**：点击开启/关闭重复设置
- **频率选择**：点击打开重复规则设置面板
- **结束条件**：点击切换结束条件类型
- **添加子任务**：点击弹出子任务输入框
- **子任务编辑**：点击子任务进入编辑模式

### 3. 重复规则设置面板

#### 主要元素
- **频率选择**：日/周/月/年单选按钮
- **高级设置**：间隔、具体日期/月份设置
- **结束条件**：选项卡切换，包含永不结束、重复次数、结束日期
- **日历预览**：底部显示重复日期预览
- **操作按钮**：取消、确定

#### 交互热区
- **频率选项**：点击切换频率类型
- **高级设置展开**：点击展开/收起高级设置
- **结束条件选项卡**：点击切换到结束条件设置
- **日历日期**：点击选择具体日期
- **确认按钮**：点击保存设置

## 五、关键交互动效设计说明

### 1. 子任务展开/收起动画
- **时长**：0.2秒
- **缓动曲线**：ease-out
- **视觉表现**：子任务列表平滑展开/收起，同时添加/移除高度
- **触发条件**：点击展开/收起按钮

### 2. 拖拽排序动画
- **时长**：0.3秒
- **缓动曲线**：ease-in-out
- **视觉表现**：拖拽时任务卡片半透明，ghost element跟随鼠标移动，释放时目标位置有轻微弹跳效果
- **触发条件**：长按拖拽手柄并移动

### 3. 子任务状态切换动画
- **时长**：0.2秒
- **缓动曲线**：ease-out
- **视觉表现**：勾选框从空心到实心的过渡动画，同时父任务进度条平滑更新
- **触发条件**：点击子任务勾选框

### 4. 重复规则设置面板动画
- **时长**：0.25秒
- **缓动曲线**：ease-out
- **视觉表现**：面板从底部滑入，高级设置区域平滑展开/收起
- **触发条件**：点击「重复」按钮打开面板，点击展开/收起高级设置

### 5. 操作反馈动画
- **时长**：0.3秒
- **缓动曲线**：ease-out
- **视觉表现**：操作成功/失败时，顶部弹出反馈提示，3秒后自动消失
- **触发条件**：完成添加、编辑、删除等操作

## 六、适配规则说明

### 1. 不同屏幕尺寸适配
- **桌面端**：完整功能展示，支持鼠标拖拽、悬停效果
- **平板端**：保持核心功能，优化触摸交互，增大点击区域
- **移动端**：简化界面，优先展示核心功能，支持触摸拖拽

### 2. 不同操作系统适配
- **Windows**：支持Ctrl快捷键，如Ctrl+N快速添加任务
- **macOS**：支持Command快捷键，如Command+N快速添加任务
- **iOS**：支持3D Touch和Haptic反馈
- **Android**：支持长按操作和振动反馈

### 3. 无障碍适配
- **屏幕阅读器**：支持所有功能的屏幕阅读器适配
- **键盘导航**：支持完全通过键盘操作所有功能
- **高对比度**：支持系统高对比度模式

## 七、用户体验验收标准

### 1. 操作路径长度
- **简单任务**：创建简单任务不超过3步
- **子任务**：创建子任务不超过4步
- **重复任务**：设置重复规则不超过5步

### 2. 学习成本
- **零教程上手**：新用户无需查看教程即可完成所有基本操作
- **功能发现**：复杂功能的入口直观，用户能够自然发现
- **操作一致性**：所有功能的操作逻辑保持一致，用户无需重新学习

### 3. 误操作率
- **拖拽操作**：误拖拽率低于5%
- **子任务管理**：子任务误操作率低于3%
- **重复规则设置**：重复规则设置错误率低于2%

### 4. 性能指标
- **响应时间**：所有操作响应时间不超过300ms
- **动画流畅度**：所有动画帧率不低于60fps
- **内存占用**：内存占用不超过100MB

## 八、总结

本交互设计方案严格遵循「无感升级、渐进式披露」的核心准则，在完全兼容原有v1.2版本交互逻辑的基础上，成功落地了子任务和重复任务两大复杂功能模块。通过直觉化的交互设计和清晰的视觉层级，确保用户在使用复杂功能时依然能够保持零学习成本，同时为专业用户提供了强大的任务管理能力。

方案中的所有设计细节都经过精心考虑，确保UI设计师和前端开发可以直接基于方案1:1还原落地，同时保持了应用的轻量易用性。通过这套设计，任务管理工具将从「简单Todo工具」成功升级为「可管理复杂任务的专业效率工具」，满足不同用户的需求。
